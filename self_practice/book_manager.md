# 실습 문제: 도서 관리 프로그램 v1.0

## 목표

Python으로 콘솔 기반 **도서 관리 프로그램**을 만드세요.

기능은 현재 Inventory 과제와 비슷합니다.

```text
도서 관리 프로그램 v1.0
---------------
[1] 도서 등록
[2] 도서 목록 조회
[3] 도서 정보 수정
[4] 도서 삭제
[0] 종료
---------------
선택:
```

---

## 1️⃣ 데이터베이스 테이블 설계

MySQL에 `book` 테이블을 만드세요.

### 테이블명

```text
book
```

### 컬럼

| 컬럼명 | 타입 | 조건 | 설명 |
|---|---|---|---|
| `id` | `INT` | `PRIMARY KEY`, `AUTO_INCREMENT` | 도서 고유 번호 |
| `title` | `VARCHAR(100)` | `NOT NULL` | 도서 제목 |
| `author` | `VARCHAR(50)` | `NOT NULL` | 저자 |
| `price` | `INT` | `NOT NULL`, 0 이상 | 도서 가격 |
| `stock` | `INT` | `NOT NULL`, 기본값 0 | 재고 수량 |
| `published_year` | `INT` | `NOT NULL` | 출판 연도 |
| `regDate` | `DATETIME` | 기본값 현재 시간 | 등록일 |

---

## 2️⃣ SQL 작성 조건

직접 `CREATE TABLE` 문을 작성하세요.

조건은 다음과 같습니다.

1. `id`는 자동 증가
2. `title`, `author`는 필수
3. `price`는 0 이상
4. `stock`은 0 이상
5. `published_year`는 1900 이상, 현재 연도 이하
6. `regDate`는 기본값 현재 시간

힌트입니다.

```sql
CREATE TABLE book (
    ...
);
```

`CHECK` 조건도 넣어보세요.

```sql
CHECK (price >= 0)
CHECK (stock >= 0)
CHECK (published_year >= 1900)
```

## 5️⃣ Pydantic 모델 조건

`book.py`에 `Book` 클래스를 만드세요.

필드는 다음과 같습니다.

```text
id
title
author
price
stock
published_year
regDate
```

검증 조건은 다음과 같습니다.

| 필드 | 조건 | 오류 메시지 |
|---|---|---|
| `title` | 빈 문자열 불가 | `ERR-2] 제목 오류: 도서 제목이 입력되지 않았습니다.` |
| `author` | 빈 문자열 불가 | `ERR-3] 저자 오류: 저자가 입력되지 않았습니다.` |
| `price` | 0 이상 | `ERR-4] 가격 오류: 가격은 0 이상이어야 합니다.` |
| `stock` | 0 이상 | `ERR-5] 재고 오류: 재고는 0 이상이어야 합니다.` |
| `published_year` | 1900 이상 | `ERR-6] 출판연도 오류: 출판연도는 1900년 이상이어야 합니다.` |

추가 조건으로 `published_year`는 현재 연도보다 크면 안 됩니다.

예를 들어 현재 연도가 2026년이면 다음과 같이 처리되어야 합니다.

```text
2027 입력 → 오류
2026 입력 → 정상
1999 입력 → 정상
1899 입력 → 오류
```

## 6️⃣ 입력 기능 요구사항

메뉴에서 `[1] 도서 등록`을 선택하면 다음 순서로 입력받으세요.

```text
도서 등록을 시작합니다.
도서 제목을 입력해주세요.
저자를 입력해주세요.
가격을 입력해주세요.
재고 수량을 입력해주세요.
출판연도를 입력해주세요.
```

입력 조건은 다음과 같습니다.

| 입력값 | 처리 |
|---|---|
| 제목 빈 값 | 오류 출력 후 메뉴로 복귀 |
| 저자 빈 값 | 오류 출력 후 메뉴로 복귀 |
| 가격 빈 값 | `-1`로 처리해서 Pydantic 검증 오류 발생 |
| 재고 빈 값 | `0`으로 처리 |
| 출판연도 빈 값 | `-1`로 처리해서 Pydantic 검증 오류 발생 |


```text

```


## 13️⃣ 이번 문제에서 꼭 연습해야 하는 포인트

| 연습 포인트 | 설명 |
|---|---|
| Pydantic 검증 | 입력값을 모델에서 검증 |
| `cursor.fetchone()` | 단건 SELECT |
| `cursor.fetchall()` | 전체 SELECT |
| `cursor.rowcount` | UPDATE/DELETE 성공 여부 |
| `return None` | 없는 id 처리 |
| `return 1 / 0` | 성공 여부 처리 |
| `input()` 빈 값 처리 | `or 0`, `or -1` 활용 |
| `try-except` | 프로그램 종료 방지 |
| `.env` | DB 접속 정보 분리 |
